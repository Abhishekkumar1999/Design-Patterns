<h3>When to use strategy design pattern</h3>
<ul>
<li>When you are multiple child class with same stragey but not same as parent means got overridden. Like, Let's assume a vehicle class having drive strategy and got overridden by many of it's child class. So, here the code duplication will happen</li>
<li>when you are writting multiple strategy with just a if else condition and when you scale the if else logic will get bloated.</li>
</ul>
